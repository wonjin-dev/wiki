# TCP/IP

> 인터넷에 연결된 서로 다른 컴퓨터들이 데이터를 주고받을 수 있도록 하는 표준 프로토콜

- 1960년대 말 ARPA에서 개발하여 ARPANET에서 사용 시작되었다.
- UNIX 기본 프로콜이다.
  - 현재 인터넷 범용 프로토콜이다.
- 응용, 전송, 인터넷, 네트워크 액세스 계층으로 이루어져 있다.

## TCP

- OSI 전송 계층에 해당한다.
- 신뢰성 있는 연결형 서비스를 제공한다.
  - 송/수신측 논리적 연결 이후 전송한다.
    - 신뢰성 보장
    - 연결 설정 지연 발생
    - 회선 이용률 저하
- 패킷의 다중화, 순서 제어, 오류 재어, 흐름 제어 기능을 제공한다.
- 스트림 전송 기능을 제공한다.

## IP

- OSI 네트워크 계층에 해당한다.
- 데이터그램을 기반으로 비연결형 서비스를 제공한다.
- `Best Effort`원칙에 따른 전송 기능을 제공한다.
  - 최선의 노력은 하짐나 전송 결과를 보장하지 않는다.
- 패킷의 분해/조립, 주소 지정, 경로 선택 기능을 제공한다.

### 응용 계층

- OSI 응용, 표현, 세션 계층
- 응용 프로그램 간의 데이터 송/수신을 제공한다.
- **_FTP_**
  > 컴퓨터와 컴퓨터(인터넷) 사이에서 파일을 주고 받을 수 있는 원격 파일 전송 프로토콜
- **_SMTP_**
  > 전자 우편을 전송하는 프로토콜
- **_TELNET_**
  - 원격 컴퓨터 사용 서비스이다.
  - 가상의 터미널 기능을 수행한다.
- **_SNMP_**
  > TCP/IP 네트워크 관리 프로토콜
- **_MQTT_**
  > 발행 - 구독 기반의 메시징 프로토콜
- **_HTTP_**
- **_DNS_**

### 전송 계층

- OSI 전송 계층
- 호스트들 간 신뢰성 있는 통신을 제공한다.
- **_TCP_**
  - 양방향 연결형 서비스를 제공한다.
  - 가상 회선 연결 형태의 서비스를 제공한다.
  - 스트림 위주의 전달을 한다.
  - 신뢰성 있는 경로를 확립하고 메시지 전송을 감독한다.
  - 순서 제어, 오류 제어, 흐름 제어 기능을 한다.
  - 패킷의 분실, 손상, 지연이나 순서가 틀린 것 등이 발생할 때 투명성이 보장된다.
  - 헤더는 기본적으로 최대 100byte를 사용할 수 있다.
- **_UDP_**
- 비연결형 서비스를 제공한다.
- TCP에 비해 상대적으로 단순한 헤더를 가진다.
  - 오버헤드가 적다.
  - 흐름 제어나 순서 제어가 없어 속도가 빠르다.
- **_RTCP_**
  > 패킷의 전송 품질을 제어하기 위한 제어 프로토콜
  - 세션에 참여한 각 참여자들에게 주기적으로 제어 정보를 전송한다.
  - 데이터 전송을 모니터링하고 최소한의 제어와 인증 기능만을 제공한다.
  - 항상 32비트의 경계로 끝난다.

### 인터넷 계층

- OSI 네트워크 계층
- 데이터 전송을 위한 주소 지정, 경로 설정을 제공한다.
- **_IMCP_**
  - IP와 조합하여 통신중에 발생하는 오류의 처리와 전송 경로 변경 등을 위한 제어 메시지를 관리하는 역할을 한다.
  - 헤더는 8바이트로 구성된다.
- **_IGMP_**
  - 멀티캐스트 그룹 유지를 위해 사용된다.
- **_ARP_**
  - 호스트의 IP 주소를 호스트와 연결된 네트워크 접속 장치의 물리적 주소로 바꾼다.
- **_RARP_**
  - ARP와 반대로 물리적 주소를 IP 주소로 변환하는 기능을 한다.
- **_IP_**

## 네트워크 액세스 계층

- OSI 데이터 링크, 물리 계층
- 실제 데이터를 송/수신하는 역할을 한다.
