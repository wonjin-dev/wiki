# IP 주소

- 총 32비트로 이루어져있다.
  - 숫자로 8비트씩 4부분
- 네트워크 부분의 길이에 따라 A~E클래스까지 총 5단계로 이루어져 있다.

```
A: 국가, 대형 통신망
B: 중대형 통신망
C: 소규모 통신망
D: 멀티 캐스트용
E: 실험적 주소. 공용되지 않음
```

## 서브네팅

> 할당된 네트워크 주소를 다시 여러 개의 작은 네트워크로 나누어 사용하는 것

- 4바이트의 IP주소 중 네트워크와 호스트 주소를 구분하기 위한 비트를 `서브넷 마스크`라고 한다.
- 서브넷 마스크는 각 클래스마다 다르게 사용된다.

## IPv6

- IPv4의 주소 부족 문제를 해결하기 위해 개발되었다.
  - 128비트의 긴 주소를 사용하여 주소 부족 문제를 해결한다.
- v4보다 비교적으로 전송 속도가 빠르다.
- 인증성, 기밀성, 데이터 무결성을 지원하여 보안 문제를 해결할 수 있다.
- 주소의 확장성, 융퉁성, 연동성이 뛰어나다.
- 실시간 흐름 제어로 향상된 멀티미디어 기능을 지원한다.
- Traffic Class, Flow Label을 이용하여 등급별, 서비스별 패킷을 구분할 수 있어 품질 보장에 용이하다.
- 패킷 크기를 확장할 수 있다.
  - 패킷 크기에 제한이 없다.
- 기본 헤더 뒤에 확장 헤더를 더함으로써 다양한 정보 저장이 가능하다.
  - 네트워크 기능 확장이 용이하다.
- 미리 예약된 알고리즘을 통해 고유성이 보장된 주소를 자동으로 구성할 수 있다.
  - 자동으로 네트워크 환경 구성이 가능하다.

### v4 &rarr; v6 전략

- 듀얼 스택
  > 모두 처리 가능하게 스택을 구성
- 터널링
  > v6 망에 인접한 v4망을 거쳐 다른 v6 망으로 통신할 때, v4 망에 터널을 만들어 v6 패킷이 통과할 수 있게 하는 방식
- 변환
  > 헤더 변환, 전송 계층 릴레이 방식, 응용 계층 게이트웨이 방식

## v6 구성

- 16비트씩 8부분. 총 128비트 구성된다.
- 각 부분을 16진수로 표현하고 `:`으로 구분한다.
- 주소 체계
  - 유니캐스트
    > 단일 송신자와 단일 수신자 (1:1)
  - 멀티 캐스트
    > 단일 송신자와 다중 수신자 (1:n)
  - 애니캐스트
    > 단일 송신자와 가장 가까이 있는 단일 수신자 (1:1)

## 도메인 네임

- 호스트 이름, 소속 기관 이름, 소속 국가명 순으로 구성된다.
  - 왼쪽에서 오른쪽으로 갈 수록 상위 도메인이다.
