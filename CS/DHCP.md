# DHCP (Dynamic Host Configuration Protocol)

네트워크에서 IP주소와 관련된 다양한 네트워크 구성 정보를 동적으로 할당하고 관리하기 위한 프로토콜이다.
장치가 네트워크에 연결될 때

- IP주소
- 서브넷 마스크
- 기본 게이트웨이
- DNS 서버 주소 및 기타 네트워크 설정

위와 같은 네트워크 구성 정보를 자동으로 획득하게 한다.

<br/>

## DHCP 서버

> 네트워크 내에서 로컬 장치에 IP주소 및 기타 네트워크 정보를 제공.

## DHCP 클라이언트

> 네트워크에 연결된 장치로, 자동으로 IP주소 및 기타 네트워크 정보를 받아오기 위해 DHCP 서버에 요청을 보낸다.

## 프로세스

1. 부팅시 DHCP 클라이언트 연결
   > 장치가 네트워크에 연결될 때, 일반적으로 **_부팅 과정_**에서 DHCP 클라이언트 소프트웨어가 실행된다.
2. DHCP 서버 탐색
   > 클라이언트는 네트워크 내에서 DHCP 서버를 찾기 위해 DHCP Discover 메시지를 브로드캐스트한다.
3. DHCP 서버 중 하나가 클라이언트의 Discover 메시지를 수신하고, 해당 클라이언트에 대한 DHCP Offer 메시지를 생성하여 제공할 IP주소와 네트워크 구성 정보를 입력한다.
4. 클라이언트 요청
   > 장치가 받은 여러 Offer 메시지 중 하나를 선택하고, 해당 서버에게 DHCP Request 메시지를 보내 선택한 IP주소를 요청한다.
5. DHCP 서버 확인
   > DHCP 서버는 클라이언트의 Request 메시지를 수신하고 IP주소를 할당한다. 그 후 DHCP Acknowledgment (ACK) 메시지를 클라이언트에게 보내어 IP주소와 구성 정보를 확인한다.
6. 클라이언트 설정
   > 클라이언트가 DHCP ACK 메시지를 받으면 할당된 IP주소 및 다른 설정 정보를 사용하여 네트워크를 구성한다.

## 장점

- IP주소 관리의 자동화
- IP주소를 중앙에서 효율적으로 관리
  > 네트워크에 새로운 장치가 추가되거나 IP주소가 변경되어도 수동으로 구성할 필요가 없음
  > IP주소 충돌이 줄음
