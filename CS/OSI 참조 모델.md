# OSI 참조 모델 (Open System Interconnection)

> 다른 시스템 간의 원활한 통신을 위해 ISO에서 제안한 통신 규약

개방형 시스템간의 데이터 통신 시 필요한 장비 및 처리 방법을 `7단계`로 표준화하였다.
<br/>

**_OSI 7계층_**
<br/>
1-3을 하위, 4-7을 상위 계층이라고 한다.

1. 물리
2. 데이터 링크
3. 네트워크
4. 전송
5. 세션
6. 표현
7. 응용

- 네트워크 계층: 패킷
- 전송 계층: 세그먼트
- 세션, 표현, 응용 계층: 메시지

## 물리 계층

- 전송에 필요한 두 장치 간의 실제 접속과 절단 등 기계적,전기적, 기능적, 절차적 특성에 대한 규칙을 정의한다.
- `비트` 데이터 단위를 사용한다.

## 데이터 링크 계층

- 두 개의 인접한 개방 시스템들 간 연결 설정과 유지 및 종료를 담당한다.
- 송/수신측 속도의 유지를 위해 흐름 제어 기능을 한다.
- 프레임의 시작과 끝을 구분하기 위해 동기화 기능을 한다.
- 오류 제어 기능을 한다.
- `프레임`을 데이터 단위로 사용한다.
- 프레임의 순서적 전송을 위해 순서적 제어 기능을 한다.

## 네트워크 계층

- 개방 시스템 간 네트워크 연결 관리 및 데이터 교환, 중계 기능을 한다.
- 네트워크 연결, 설정, 유지, 해제 기능을 한다.
- 발신자의 목적지와 논리 주소가 추가된 패킷을 최종 목적지까지 전달하는 책임을 가진다.
- 경로 설정, 데이터 교환 및 중계, 트래픽 제어, 패킷 정보 전송을 수행한다.

## 전송 계층

- 논리적 안정과 균일한 데이터 전송 서비스를 제공하여 E2E간 투명한 데이터 전송을 가능하게 한다.
- 하위 계층과 상위 계층의 인터페이스를 담당한다.
- E2E간 전송 연결 설정, 데이터 전송, 연결 해제 기능을 한다.
- 주소 설정, 다중화, 오류 제어, 흐름 제어 역할을 수행한다.
- `TCP`, `UDP`등의 표준이 있다.

## 세션 계층

- 송/수신 간 관련성을 유지하고 대화 제어를 담당한다.
- `토큰`을 사용하여 송수신측간 데이터 전송, 연결 해제, 동기 처리를 수행한다.
- 대화 동기를 위해 전송하는 정보의 일정한 부분에 체크점을 두어 정보의 수신 상태를 확인한다.
- 동기점은 오류가 있는 데이터의 회복을 위해 사용한다.

## 표현 계층

- 응용 계층으로부터 받은 데이터를 세션 계층에 보내기 전에 통신에 적당한 형태로 변환한다.
- 세션 계층에서 받은 데이터는 응용 계층에 맞게 변환하는 기능을 한다.
- 서로 다른 데이터 표현 형태를 갖는 시스템 간의 상호 접속을 위해 필요한 계층이다.
- 코드 변환, 데이터 암호화, 데이터 압축, 구문 검색, 정보 형식 변환, 문맥 관리 기능을 한다.

## 응용 계층

- 사용자가 OSI 환경에 접근할 수 있도록 서비스를 제공한다.
